    13  // LogsSchemaCategoryMapper Use the Schema Category Mapper to categorize log event into enum fields.
    14  // In the case of OCSF, they can be used to map sibling fields which are composed of an ID and a name.
    15  //
    16  // **Notes**:
    17  //

    20  //     Wildcards can also be used inside your query.
    21  //   - Categories are executed in order and processing stops at the first match.
    22  //     Make sure categories are properly ordered in case a log could match multiple queries.
    23  //   - Sibling fields always have a numerical ID field and a human-readable string name.
    24  //   - A fallback section handles cases where the name or ID value matches a specific value.
